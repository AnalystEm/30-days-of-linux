# Day 04 - Linux Kernel and Resource Management

## Objective

My goal for today was to understand the Linux kernel more clearly, learn its role in the Linux operating system, and study how it manages system resources and core subsystems.

---

## What I Learned

- The Linux kernel is the core part of the Linux operating system that connects software with hardware.
- The kernel manages system resources and helps applications function properly.
- The Linux kernel acts as a bridge between hardware and software.
- The kernel is open-source and has grown through community contributions.
- In a general-purpose computer, the kernel helps manage multiple running processes efficiently.
- The kernel allocates CPU time to processes.
- The kernel manages system memory.
- The kernel controls access to hardware devices.
- The kernel prevents conflicts between processes.
- The kernel provides virtual resources to applications.
- The Linux kernel is made up of several core subsystems that work together to manage system resources.
- Important kernel subsystems include the Process Scheduler, Memory Management Unit, Virtual File System, Networking Unit, and Inter-Process Communication Unit.

## Key Takeaways

- The kernel is the central part of Linux and plays a major role in how the operating system works.
- It is responsible for managing resources and coordinating communication between software and hardware.
- Resource management is one of the most important functions of the kernel.
- The Linux kernel is made up of several subsystems that each handle a different part of system management.
- Understanding the kernel gives me a deeper understanding of how Linux works internally.
