# Day 06 - Shell Types and Common Shell Programs in Linux

## Objective

My goal for today was to understand the different types of shells in Linux, learn the difference between command line and graphical shells, and become familiar with common shell programs used in Linux systems.

---

## What I Learned

- Shells in Linux are broadly classified into two categories: command line shells and graphical shells.
- A command line shell allows users to interact with the system by typing human-readable commands through a terminal program.
- In Linux and macOS, users usually access the shell through a terminal application, while Windows uses programs such as Command Prompt.
- Commands entered into the command line shell are executed and the results are displayed back to the user in the terminal.
- Command line interaction can be challenging for beginners because it requires remembering commands, but it is powerful and useful for automation.
- In Linux and macOS, saved command files are known as shell scripts.
- A graphical shell allows users to interact with the system through a graphical user interface instead of typing commands.
- Graphical shells support actions such as window-based program management, drag-and-drop file handling, mouse-driven navigation, and visual system settings configuration.
- Linux supports multiple shell programs that perform similar core tasks but differ in syntax, features, and built-in functionality.
- Bash, or Bourne Again Shell, is one of the most widely used Linux shells and is commonly the default shell in many Linux systems.
- Csh, or C Shell, has syntax and usage that are similar to the C programming language.
- Ksh, or Korn Shell, also played an important role in the development of POSIX shell standards.

---

## Key Takeaways

- Linux offers more than one type of shell, and each serves as a way for users to interact with the operating system.
- The command line shell and graphical shell both help users work with the system, but they do so in very different ways.
- Command line shells are powerful and flexible, especially for automation and scripting.
- Graphical shells are easier for many beginners because they allow interaction through windows, icons, and menus.
- Knowing common Linux shell programs like Bash, Csh, and Ksh gives me a better understanding of the Linux command environment.
