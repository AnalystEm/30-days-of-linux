 14 - File Management in Linux

## Objective

My goal for today was to understand file management in Linux, learn the main file types in Linux, and study common file operations such as listing files, creating files, viewing file contents, and copying files.

---

## What I Learned

- File management in Linux involves handling files and directories through operations such as creation, modification, organization, and access control.
- Linux treats many things as files, including devices and system configurations.
- Linux file management relies on commands such as `ls`, `touch`, `cat`, `cp`, `mv`, `rm`, and `chmod`.
- Linux groups files into three main types: general files, directory files, and device files.
- General files store user data such as text, images, scripts, programs, and binaries.
- Directory files act as containers that organize files and other directories hierarchically.
- Device files represent hardware devices and are mostly found in the `/dev/` directory.
- The `ls` command is used to list files and directories in the current location.
- The `ls -l` command gives a detailed listing that includes file permissions, owner, group, file size, and modification date.
- The `touch` command is used to create a new empty file.
- If the file already exists, `touch` can update its timestamp without changing its content.
- The `cat` command is used to display the contents of a file in the terminal.
- The `cp` command is used to copy a file from one location to another.

---

## Key Takeaways

- File management is a core part of working in Linux because almost everything is handled through files and directories.
- Linux has different file types, and each one serves a different purpose in the system.
- Commands like `ls` and `ls -l` help me inspect files and understand their details.
- Commands like `touch`, `cat`, and `cp` are useful for creating, viewing, and copying files.
- Understanding these basics gives me a stronger foundation for handling files confidently in Linux.
